# Функциональные требования

## 1. Управление транзакциями

| ID | Требование | Приоритет |
|----|------------|-----------|
| TR-01 | Пользователь может создать транзакцию с указанием даты, суммы, категории, описания, счёта и конверта | high |
| TR-02 | Пользователь может указать тип транзакции: доход, расход, перевод | high |
| TR-03 | Пользователь может выбрать счёт для транзакции | high |
| TR-04 | Пользователь может редактировать существующую транзакцию | high |
| TR-05 | Пользователь может удалить транзакцию | high |
| TR-06 | Пользователь может создать регулярную транзакцию с периодичностью (ежедневно, еженедельно, ежемесячно, ежегодно) | medium |
| TR-07 | Система автоматически создаёт регулярные транзакции в указанные даты | medium |
| TR-08 | Пользователь может просмотреть список всех транзакций с фильтрацией (по дате, категории, счёту, конверту, типу, описанию) | high |
| TR-09 | Пользователь может указать конверт (ХаниМани, Резервы, Фонд, Цель) для транзакции | high |
| TR-10 | Пользователь может ввести сумму и описание через natural language: `5*250 яблоки` = 5×250, текст в `(скобках)` игнорируется | high |
| TR-11 | Пользователь может создать транзакцию в режиме Факт (реальная) или План (запланированная) | high |
| TR-12 | Пользователь может копировать транзакцию перетаскиванием с зажатым Alt/Ctrl в календаре | medium |

## 2. Календарь

| ID | Требование | Приоритет |
|----|------------|-----------|
| CA-01 | Пользователь видит календарь на месяц с отображением транзакций по дням (недели × дни) | high |
| CA-02 | Пользователь может переключаться между месяцами | high |
| CA-03 | Пользователь может кликнуть на день, чтобы увидеть детальные транзакции | high |
| CA-04 | Пользователь может быстро добавить транзакцию из календаря | high |
| CA-05 | В календаре отображается сводка: доходы, расходы, разница за месяц, % не потрачено | high |
| CA-06 | На всех страницах в хедере отображается Net Worth (ХаниМани + Резервы + Фонды + Цели) | high |
| CA-07 | Пользователь может открыть панель статистики (ALT+Z) с расходами и доходами по категориям | high |

## 3. Категории

| ID | Требование | Приоритет |
|----|------------|-----------|
| CT-01 | Система предоставляет предустановленный набор категорий (~50 шт) | high |
| CT-02 | Пользователь может создать свою категорию | high |
| CT-03 | Пользователь может редактировать категорию | medium |
| CT-04 | Пользователь может удалить категорию (не используется в транзакциях) | medium |
| CT-05 | Категории иерархические: группа/подкатегория (разделитель `/`) | medium |
| CT-06 | Каждая категория имеет тип: доход или расход | high |
| CT-07 | В редакторе категорий отображается счётчик использований категории | medium |

## 4. Счета

| ID | Требование | Приоритет |
|----|------------|-----------|
| AC-01 | Пользователь может создать счёт (наличные, кредитная карта, chequing, savings, investment) | high |
| AC-02 | Пользователь может редактировать счёт | high |
| AC-03 | Пользователь может удалить счёт (при отсутствии транзакций) | medium |
| AC-04 | Отображается текущий баланс по каждому счёту | high |
| AC-05 | Для кредитных карт отображается доступный баланс (лимит - текущий) | high |
| AC-06 | Пользователь может исключить счёт из общего баланса | medium |
| AC-07 | Пользователь может просмотреть историю операций по счёту | medium |
| AC-08 | Счета группируются по разделам: Favorites, Investments, Hidden | medium |

## 5. Конверты (Envelopes)

| ID | Требование | Приоритет |
|----|------------|-----------|
| EN-01 | Система предоставляет 4 типа конвертов: ХаниМани, Резервы, Фонды, Цели | high |
| EN-02 | Пользователь может создать конверт любого типа | high |
| EN-03 | Пользователь может редактировать конверт | high |
| EN-04 | Пользователь может удалить конверт | medium |
| EN-05 | Для Фондов отображается баланс и сумма фонда | high |
| EN-06 | Для Целей отображается баланс и целевая сумма | high |
| EN-07 | Баланс конверта автоматически обновляется при транзакциях | high |
| EN-08 | Пользователь видит сводку по всем конвертам на странице «Счета и Конверты» | high |
| EN-09 | Конверты можно скрывать из общего списка | medium |

## 6. Аналитика

| ID | Требование | Приоритет |
|----|------------|-----------|
| AN-01 | Пользователь видит круговую диаграмму расходов по категориям за период | high |
| AN-02 | Пользователь может раскрыть подкатегории внутри группы расходов | medium |
| AN-03 | Пользователь видит доходы по категориям с процентом от общего дохода | high |
| AN-04 | Пользователь может выбрать период для отчёта | high |
| AN-05 | Пользователь видит сумму доходов, расходов и разницу за период | high |
| AN-06 | Пользователь может сравнить доходы/расходы за два периода | medium |
| AN-07 | Пользователь видит отчёт Net Worth (все счета и конверты) | medium |

## 7. Аутентификация

| ID | Требование | Приоритет |
|----|------------|-----------|
| AU-01 | Пользователь может зарегистрироваться (email + пароль) | high |
| AU-02 | Пользователь может войти в систему | high |
| AU-03 | Пользователь может выйти из системы | high |
| AU-04 | Неаутентифицированный пользователь не имеет доступа к данным | high |

## 8. Горячие клавиши

| ID | Требование | Приоритет |
|----|------------|-----------|
| HK-01 | Ctrl+Enter — сохранить открытую транзакцию | high |
| HK-02 | Esc — закрыть любой модал | high |
| HK-03 | Alt+1 — Счета и Конверты, Alt+2 — Календарь, Alt+3 — Список, Alt+4 — Отчёты | high |
| HK-04 | Alt+← — предыдущий месяц, Alt+→ — следующий месяц, Alt+↑ — текущий месяц | medium |
| HK-05 | Alt+Z — показать/скрыть статистику по категориям | medium |
| HK-06 | Alt+X — показать прогноз Резервов (Net Worth popover) | medium |
