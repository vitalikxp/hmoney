# Техническое задание: HoneyMoney Lite

## 1. Назначение

Разработать веб-приложение (SPA + PWA) для ручного учёта личных финансов — упрощённый аналог HoneyMoney (https://hmbee.ru).

Приложение предназначено для одного пользователя (личное использование), с возможностью расширения на семью.

## 2. Целевая аудитория

- Люди, которые хотят вести учёт личных финансов вручную
- Люди, ценящие осознанное отношение к деньгам
- Пользователи, не желающие подключать банковские счета к приложениям

## 3. Основные функции (MVP)

### 3.1 Управление транзакциями
- Создание, редактирование, удаление транзакций
- Поля: дата, сумма, категория, описание, счёт, конверт
- Типы: доход, расход, перевод; неподтверждённые — статус (фильтр в списке)
- Режимы: Факт (реальная) и План (запланированная)
- Natural language input: `5*250 яблоки (комментарий)` — парсинг количества × цена
- Поддержка регулярных (recurring) транзакций
- Фильтрация по счетам, конвертам, типам, категориям, описанию

### 3.2 Календарь
- Просмотр транзакций в календарном формате (недели × дни)
- Быстрое добавление транзакции кликом на день
- Сводка доходов/расходов/разницы за месяц + % не потрачено
- Net Worth в хедере на всех страницах (ALT+X)
- Статистика расходов/доходов по категориям (ALT+Z) с детализацией по подкатегориям
- Drag-and-drop копирование транзакций (зажат Alt/Ctrl)

### 3.3 Категории
- Иерархические категории (группа/подкатегория через `/`)
- Предустановленный набор категорий (~50 шт)
- Пользовательские категории
- Счётчик использований категории
- Редактор категорий (модальное окно)

### 3.4 Счета
- Типы: кредитные карты, chequing, savings, investment, cash
- Отслеживание баланса по каждому счёту
- Кредитный лимит для кредитных карт
- Флаг исключения из общего баланса
- Группировка: Favorites, Investments, Hidden

### 3.5 Конверты (Envelopes)
Четыре типа конвертов:
1. **ХаниМани** — свободные деньги на текущие траты
2. **Резервы** — зарезервировано на плановые расходы
3. **Фонды** — целевые фонды (с балансом и суммой фонда)
4. **Цели** — долгосрочные сбережения (с балансом и целевой суммой)

### 3.6 Базовая аналитика
- Расходы по категориям и подкатегориям
- Доходы по категориям с процентами
- Доходы/расходы/разница за период
- Сравнение периодов
- Net Worth отчёт

### 3.7 Горячие клавиши
- Ctrl+Enter — сохранить транзакцию
- Esc — закрыть модал
- Alt+1..4 — навигация по разделам
- Alt+←/→ — переключение месяца
- Alt+↑ — текущий месяц
- Alt+Z — статистика
- Alt+X — Net Worth popover

## 4. Технические требования

### 4.1 Платформа
- **Хостинг**: GitHub Pages
- **Формат**: SPA (Single Page Application)
- **PWA**: поддержка офлайн-режима

### 4.2 Фронтенд
- **Язык**: TypeScript
- **Фреймворк**: React 18+
- **Бандлер**: Vite
- **Стили**: Tailwind CSS
- **Роутинг**: React Router

### 4.3 Бекенд и данные
- **База данных**: Firebase Firestore (free tier)
- **Аутентификация**: Firebase Auth
- **Офлайн**: кэширование данных локально (PWA + Firestore offline persistence)

### 4.4 Ограничения free tier Firebase
- 1 GB хранимых данных
- 10 GB/мес загрузок
- 50k reads/day, 20k writes/day
- 10 MB/request

## 5. Ограничения по реализации

- Только ручной ввод (нет автоматической синхронизации с банками)
- Однопользовательский режим (MVP)
- Локализация — русский язык (MVP)
- Валюта — RUB (MVP)

## 6. План работ

1. **Этап 1: Инициализация проекта** — Vite + React + Tailwind + Firebase
2. **Этап 2: Аутентификация** — регистрация, вход, защита маршрутов
3. **Этап 3: Категории** — CRUD категорий
4. **Этап 4: Счета и Конверты** — CRUD счетов и 4 типа конвертов
5. **Этап 5: Транзакции** — CRUD транзакций, natural language input, Факт/План
6. **Этап 6: Календарь** — календарный просмотр, статистика
7. **Этап 7: Аналитика** — отчёты по месяцам, net worth
8. **Этап 8: PWA** — офлайн-режим, иконки, манифест
9. **Этап 9: Полировка** — горячие клавиши, UI/UX, тесты, деплой

## Вопросы и противоречия

- **План vs Факт**: как План-транзакции влияют на баланс счетов? В референсе — не влияют, но в модели нет флага `affectsBalance`.
- **Регулярные транзакции**: создаются на клиенте (CRON в браузере?) или cloud functions? В MVP — ручное создание через шаблон.
- **Категории ~50 шт**: где хранить предустановленный список? Seed в Firestore при регистрации или константы в коде?
