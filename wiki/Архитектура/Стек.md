# Обоснование стека

## React 19 + Vite

| Критерий | Выбор | Альтернативы |
|----------|-------|-------------|
| Сборка | Vite | CRA (устарел), Next.js (избыточен для SPA) |
| UI | React 19 | Svelte, Solid (меньше экосистема) |
| Роутинг | React Router 7 | TanStack Router |
| Язык | TypeScript | JavaScript (меньше safety) |

**Почему не Next.js**: нам не нужен SSR, приложение полностью клиентское.
GitHub Pages не поддерживает SSR. SPA с Vite — оптимальный выбор.

## Tailwind CSS v4

- Быстрая разработка UI без кастомных CSS-файлов
- Отличная интеграция с React
- Маленький итоговый бандл (purge)
- v4 — конфигурация через CSS (`@theme`), без tailwind.config.js

## GitHub Pages + Actions (CI/CD)

- Хостинг статики: GitHub Pages (бесплатно)
- Кастомный домен: https://money.vitalik.dev (CNAME в репозитории)
- CI/CD: GitHub Actions — автоматическая сборка и деплой при пуше в `master`
- Воркфлоу: `actions/checkout` → `setup-node` → `npm ci` → `npm run build` → `actions/deploy-pages`
- Превью продакшен-сборки локально: `npm run preview`

## Firebase

| Сервис | Назначение | Бесплатный лимит |
|--------|-----------|-----------------|
| Firestore | База данных | 1GB, 50k reads/day, 20k writes/day |
| Auth | Аутентификация | Безлимит (email/password) |
| Hosting | (не нужен — GitHub Pages) | — |

**Альтернатива**: Supabase — тоже имеет бесплатный tier, но требует больше конфигурации.
Firestore проще для старта: встроенная офлайн-персистентность, реальное время, меньше бойлерплейта.

## Zustand (state manager)

- Минимальный API (нет reducers, actions, dispatchers)
- Не требует Provider
- Работает вне React-компонентов
- TypeScript-first

## Vitest + Testing Library (unit-тестирование)

| Инструмент | Назначение |
|-----------|-----------|
| Vitest | Test runner (нативный Vite-плагин, не требует Babel) |
| @testing-library/react | Рендер компонентов в jsdom |
| @testing-library/jest-dom | Матчеры (toBeInTheDocument, toHaveTextContent) |
| @testing-library/user-event | Симуляция событий (click, type, tab) |
| jsdom | Браузерное окружение для Node.js |

- **Конфиг**: `vitest.config.ts` (jsdom, setup-файл)
- **Setup**: `src/test/setup.ts` (jest-dom + глобальный afterEach(cleanup))
- **137 тестов** (16 файлов): stores, components, pages
- **CI**: `npm test` перед `npm run build` — если тесты падают, деплой блокируется
- **Static analysis**: только `tsc -b` (strict mode), ESLint/Biome не используются
- **Паттерн моков**: Firebase модули на уровне `vi.mock`, Zustand stores через `vi.hoisted()`

## PWA (план)

Офлайн-режим через `vite-plugin-pwa` — запланирован на Этап 7, не реализован в текущей версии.
Текущая мобильная поддержка: адаптивный Layout (mobile bottom nav через Tailwind `md:hidden`).
