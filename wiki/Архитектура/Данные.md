# Модели данных

## Firestore коллекции

### `users/{userId}`
```typescript
interface UserProfile {
  email: string;
  createdAt: Timestamp;
  updatedAt: Timestamp;
}
```

**Планируется:** `displayName`, `currency`

### `users/{userId}/accounts/{accountId}`
```typescript
interface Account {
  name: string;             // "Наличные", "Тинькофф", etc.
  balance: number;          // текущий баланс в целых рублях
  creditLimit?: number;     // для кредитных карт (в рублях); если есть — тип = card
  icon?: string;            // emoji, по умолчанию '💳'
  includeInBalance: boolean; // участвует в общем балансе
  currency: string;         // 'RUB' (всегда, post-MVP — мультивалютность)
  sortOrder: number;        // хранится, но сортировка по createdAt
  group: 'favorites' | 'investments' | 'hidden' | 'default';
  createdAt: Timestamp;
  updatedAt: Timestamp;
}
```

Тип счёта не хранится — выводится:
- `creditLimit` присутствует → `card`
- `group === 'investments'` → `investment`
- иначе → `cash`

**Планируется:** `color`

### `users/{userId}/envelopes/{envelopeId}`
```typescript
interface Envelope {
  name: string;             // "Резервы", "Отпуск", etc.
  balance: number;          // текущий баланс в целых рублях
  isGoal: boolean;          // true — конверт-цель (target обязателен)
  target?: number;          // целевая сумма (в рублях); обязательна при isGoal=true
  icon?: string;            // emoji, по умолчанию '✉️'
  isBuiltIn?: boolean;      // true только для Резервов (нельзя удалить)
  sortOrder: number;        // хранится, но сортировка по createdAt
  isHidden: boolean;        // скрыт ли из общего списка
  createdAt: Timestamp;
  updatedAt: Timestamp;
}
```

Поле `type` отсутствует. Системный конверт идентифицируется через `isBuiltIn: true`.

#### ХаниМани — вычисляемое значение (не документ)

ХаниМани не хранится в Firestore. Рассчитывается на клиенте при каждом рендере:

```
ХаниМани = Σ(account.balance | includeInBalance=true)
           − Σ(envelope.balance | isBuiltIn=true)    // Резервы
           − Σ(envelope.balance | isBuiltIn≠true)    // пользовательские конверты
```

Изменяется автоматически при любом изменении счетов или конвертов. Для транзакций без конкретного конверта: `envelopeId: null` = ХаниМани.

#### Виды конвертов (Firestore-документы)

**Системный** (ровно 1, неудаляемый):
- **Резервы** — `isBuiltIn: true`, `isGoal: false`. Создаётся автоматически при регистрации.

**Пользовательские** (0..20 суммарно):
- Обычный конверт — `isGoal: false`, `target` не задан
- Конверт-цель — `isGoal: true`, `target` обязателен; UI показывает прогресс-бар

Лимит: **20 пользовательских конвертов**. Проверяется в `envelopeStore.createEnvelope`.

#### Палитра иконок
Первая иконка `✉️` — дефолт для новых конвертов (`DEFAULT_ICON`). Полный список — `ICONS` в `src/components/envelopes/constants.ts`.

#### Прогноз Резервов (вычисляемый)
Не хранится в Firestore — рассчитывается на клиенте. Горизонт: до **12 месяцев вперёд** (аналогично оригинальному HM).
```
дефицит/профицит = balance.reserves
  + SUM(transactions.plan.income)
  - SUM(transactions.plan.expense)
  - SUM(plannedTransfers.toReserves)
  + SUM(plannedTransfers.fromReserves)
```

> ⚠️ **Не реализовано.** Ниже — проектная модель.

### `users/{userId}/transactions/{transactionId}`
```typescript
type TransactionType = 'income' | 'expense' | 'transfer';

type TransactionStatus = 'confirmed' | 'unconfirmed';

type TransactionMode = 'actual' | 'plan';

interface TransactionItem {
  quantity: number;         // количество (из natural language парсинга)
  unitPrice: number;        // цена за единицу (в рублях)
  description: string;      // текст позиции
}

interface Transaction {
  date: Timestamp;              // дата транзакции
  amount: number;               // итоговая сумма в рублях
  type: TransactionType;
  mode: TransactionMode;        // Факт или План
  status: TransactionStatus;
  category: string;             // свободная строка, не справочник
  accountId: string;            // ссылка на счёт
  toAccountId?: string;         // для переводов — целевой счёт
  envelopeId?: string;          // ссылка на конверт
  description: string;          // описание (из natural language)
  items?: TransactionItem[];    // парсенные позиции из natural language
  isRecurring: boolean;
  recurringRule?: {             // если регулярная
    frequency: 'daily' | 'weekly' | 'monthly' | 'yearly';
    interval: number;           // каждые N дней/недель/etc
    endDate?: Timestamp;        // дата окончания
  };
  parentTransactionId?: string; // для регулярных — ссылка на шаблон
  createdAt: Timestamp;
  updatedAt: Timestamp;
}
```

### Natural language парсинг

Формат ввода: `5*250 яблоки (комментарий)`

- `5*250` = 5 единиц × 250 = 1250
- `яблоки` = описание
- `(комментарий)` = игнорируется, заметка
- Несколько позиций через запятую: `3*150 кофе, 2*200 пирожки`

## Индексы Firestore

1. `transactions` — по `date` (desc) + `accountId`
2. `transactions` — по `category` + `date`
3. `transactions` — по `isRecurring` + `date`
4. `transactions` — по `envelopeId` + `date`
5. `transactions` — по `type` + `date`
6. `transactions` — по `mode` + `date`

## Хранение сумм

Все суммы хранятся в **целых рублях** (шаг 1). Копейки не используются.
Отображение: число без дробной части.

⚠️ После реализации транзакций — убедиться, что `amount`, `TransactionItem.unitPrice` и `Envelope.balance` не записаны «в копейках».
