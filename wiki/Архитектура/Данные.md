# Модели данных

### `users/{userId}/categories/{categoryId}`
```typescript
interface Category {
  name: string;             // "food/groceries-household" — иерархия через /
  type: 'income' | 'expense';
  parentId?: string;        // ссылка на родительскую категорию

  color?: string;           // hex color
  sortOrder: number;
  usageCount: number;       // счётчик использований

  updatedAt: Timestamp;

```

### `users/{userId}/accounts/{accountId}`
```typescript
interface Account {
  name: string;             // "Наличные", "Тинькофф", etc.
  type: 'cash' | 'card' | 'chequing' | 'savings' | 'investment';
  balance: number;          // текущий баланс в копейках
  creditLimit?: number;     // для кредитных карт (в копейках)
  includeInBalance: boolean; // исключён ли из общего баланса
  currency: string;         // 'RUB'
  icon?: string;
  color?: string;
  sortOrder: number;
  group: 'favorites' | 'investments' | 'hidden' | 'default';
  createdAt: Timestamp;
  updatedAt: Timestamp;
}
```

### `users/{userId}/envelopes/{envelopeId}`
```typescript
type EnvelopeType = 'spending' | 'reserve' | 'fund' | 'goal';

interface Envelope {
  name: string;             // "ХаниМани", "Резервы", "Emergency Fund"
  type: EnvelopeType;
  balance: number;          // текущий баланс в копейках
  target?: number;          // целевая сумма (для фондов и целей)
  icon?: string;
  color?: string;
  sortOrder: number;
  isHidden: boolean;        // скрыт ли из общего списка
  createdAt: Timestamp;
  updatedAt: Timestamp;
}
```

#### Типы конвертов
| Тип | Назначение | Target |
|-----|-----------|--------|
| `spending` | ХаниМани — свободные деньги | нет |
| `reserve` | Резервы — на плановые расходы | нет |
| `fund` | Фонды — целевые (Emergency Fund) | optional |
| `goal` | Цели — долгосрочные (TFSA, RRSP) | required |

### `users/{userId}/transactions/{transactionId}`
```typescript
type TransactionType = 'income' | 'expense' | 'transfer';

type TransactionStatus = 'confirmed' | 'unconfirmed';

type TransactionMode = 'actual' | 'plan';

interface TransactionItem {
  quantity: number;         // количество (из natural language парсинга)
  unitPrice: number;        // цена за единицу (в копейках)
  description: string;      // текст позиции
}

interface Transaction {
  date: Timestamp;              // дата транзакции
  amount: number;               // итоговая сумма в копейках
  type: TransactionType;
  mode: TransactionMode;        // Факт или План
  status: TransactionStatus;
  categoryId: string;           // ссылка на категорию
  accountId: string;            // ссылка на счёт
  toAccountId?: string;         // для переводов — целевой счёт
  envelopeId?: string;          // ссылка на конверт
  description: string;          // описание (из natural language)
  items?: TransactionItem[];    // парсенные позиции из natural language
  isRecurring: boolean;
  recurringRule?: {             // если регулярная
    frequency: 'daily' | 'weekly' | 'monthly' | 'yearly';
    interval: number;           // каждые N дней/недель/etc
    endDate?: Timestamp;        // дата окончания
  };
  parentTransactionId?: string; // для регулярных — ссылка на шаблон
  createdAt: Timestamp;
  updatedAt: Timestamp;
}
```

### Natural language парсинг

Формат ввода: `5*250 яблоки (комментарий)`

- `5*250` = 5 единиц × 250 = 1250
- `яблоки` = описание
- `(комментарий)` = игнорируется, заметка
- Несколько позиций через запятую: `3*150 кофе, 2*200 пирожки`

## Индексы Firestore

1. `transactions` — по `date` (desc) + `accountId`
2. `transactions` — по `categoryId` + `date`
3. `transactions` — по `isRecurring` + `date`
4. `transactions` — по `envelopeId` + `date`
5. `transactions` — по `type` + `date`
6. `transactions` — по `mode` + `date`

## Хранение сумм

Все суммы хранятся в **копейках** (целые числа) чтобы избежать проблем с плавающей точкой.
Отображение: `amount / 100` с двумя знаками после запятой.

## Мультивалютность

В MVP только RUB. В будущем:
- У каждого счёта своя валюта
- Базовая валюта пользователя для отчётов
- Курсы валют (периодическое обновление)
- Конвертация сумм в базовую валюту для отображения

## Вопросы и противоречия

- **Transfer без модели**: для перевода нужен `toAccountId`. Добавлен, но нет логики атомарного списания/зачисления с двух счетов.
- **Индексы Firestore**: составные индексы (date + accountId и т.д.) требуют создания в Firebase console — не создадутся автоматически.
- **usageCount**: как обновляется — триггером из клиента? Или через cloud function? При удалении транзакции счётчик должен уменьшаться.
