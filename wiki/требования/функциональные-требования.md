# Функциональные требования

## 1. Управление транзакциями

| ID | Требование | Приоритет |
|----|------------|-----------|
| TR-01 | Пользователь может создать транзакцию с указанием даты, суммы, категории и описания | high |
| TR-02 | Пользователь может указать тип транзакции: доход или расход | high |
| TR-03 | Пользователь может выбрать счёт для транзакции | high |
| TR-04 | Пользователь может редактировать существующую транзакцию | high |
| TR-05 | Пользователь может удалить транзакцию | high |
| TR-06 | Пользователь может создать регулярную транзакцию с периодичностью (ежедневно, еженедельно, ежемесячно, ежегодно) | medium |
| TR-07 | Система автоматически создаёт регулярные транзакции в указанные даты | medium |
| TR-08 | Пользователь может просмотреть список всех транзакций с фильтрацией (по дате, категории, счёту) | high |

## 2. Календарь

| ID | Требование | Приоритет |
|----|------------|-----------|
| CA-01 | Пользователь видит календарь на месяц с отображением транзакций по дням | high |
| CA-02 | Пользователь может переключаться между месяцами | high |
| CA-03 | Пользователь может кликнуть на день, чтобы увидеть детальные транзакции | high |
| CA-04 | Пользователь может быстро добавить транзакцию из календаря | high |
| CA-05 | В календаре отображается сумма доходов и расходов за день | high |

## 3. Категории

| ID | Требование | Приоритет |
|----|------------|-----------|
| CT-01 | Система предоставляет предустановленный набор категорий | high |
| CT-02 | Пользователь может создать свою категорию | high |
| CT-03 | Пользователь может редактировать категорию | medium |
| CT-04 | Пользователь может удалить категорию (не используется в транзакциях) | medium |
| CT-05 | Категории могут быть вложенными (группа → подкатегория) | medium |
| CT-06 | Каждая категория имеет тип: доход или расход | high |

## 4. Счета (Envelopes)

| ID | Требование | Приоритет |
|----|------------|-----------|
| EN-01 | Пользователь может создать счёт (наличные, карта, сбережения) | high |
| EN-02 | Пользователь может редактировать счёт | high |
| EN-03 | Пользователь может удалить счёт (при отсутствии транзакций) | medium |
| EN-04 | Отображается текущий баланс по каждому счёту | high |
| EN-05 | Система автоматически обновляет баланс счёта при создании транзакции | high |
| EN-06 | Пользователь может просмотреть историю операций по счёту | medium |

## 5. Аналитика

| ID | Требование | Приоритет |
|----|------------|-----------|
| AN-01 | Пользователь видит круговую диаграмму расходов по категориям за период | high |
| AN-02 | Пользователь видит динамику баланса по месяцам | medium |
| AN-03 | Пользователь может выбрать период для отчёта | high |
| AN-04 | Пользователь видит сумму доходов, расходов и разницу за период | high |
| AN-05 | Пользователь может сравнить два периода | low |

## 6. Аутентификация

| ID | Требование | Приоритет |
|----|------------|-----------|
| AU-01 | Пользователь может зарегистрироваться (email + пароль) | high |
| AU-02 | Пользователь может войти в систему | high |
| AU-03 | Пользователь может выйти из системы | high |
| AU-04 | Неаутентифицированный пользователь не имеет доступа к данным | high |
