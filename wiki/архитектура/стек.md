# Обоснование стека

## React + Vite

| Критерий | Выбор | Альтернативы |
|----------|-------|-------------|
| Сборка | Vite | CRA (устарел), Next.js (избыточен для SPA) |
| UI | React | Svelte, Solid (меньше экосистема) |
| Язык | TypeScript | JavaScript (меньше safety) |

**Почему не Next.js**: нам не нужен SSR, приложение полностью клиентское.
GitHub Pages не поддерживает SSR. SPA с Vite — оптимальный выбор.

## Tailwind CSS

- Быстрая разработка UI без кастомных CSS-файлов
- Отличная интеграция с React
- Маленький итоговый бандл (purge)

## Firebase

| Сервис | Назначение | Бесплатный лимит |
|--------|-----------|-----------------|
| Firestore | База данных | 1GB, 50k reads/day, 20k writes/day |
| Auth | Аутентификация | Безлимит (email/password) |
| Hosting | (не нужен — GitHub Pages) | — |

**Альтернатива**: Supabase — тоже имеет бесплатный tier, но требует больше конфигурации.
Firestore проще для старта: встроенная офлайн-персистентность, реальное время, меньше бойлерплейта.

## Zustand (state manager)

- Минимальный API (нет reducers, actions, dispatchers)
- Не требует Provider
- Работает вне React-компонентов
- TypeScript-first
