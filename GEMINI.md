# Инструкции по проекту: hmoney

Этот проект представляет собой приложение для отслеживания личных финансов ("hmoney"), упрощенный клон HoneyMoney. Оно разработано как современное React SPA с упором на высококонтрастный дизайн и тщательное тестирование.

## Основные требования

### Язык проекта
- **Основной язык:** Русский язык является обязательным для всей документации, описания тестов и взаимодействия с AI. 
- **Планы:** Все планы, составляемые агентами и Gemini, должны быть на русском языке. 
- **Технический язык:** Английский язык используется исключительно для написания кода, имен переменных и технических терминов.

### Контроль версий и рабочий процесс
- **Ветвление:** Вся разработка ведется в ветке `dev`.
- **Коммиты:** Никогда не делайте коммиты и не индексируйте изменения без явного разрешения пользователя.
- **Деплой:** Слияние с веткой `master` и деплой выполняются ТОЛЬКО по явному запросу пользователя ("deploy", "merge to master").

### Документация и Wiki (3-уровневая структура)
- **Источники (`wiki/Источники/`):** Только для чтения для AI. Содержит оригинальные исследования и документы.
- **Wiki (`wiki/Требования/`, `wiki/Архитектура/`, `wiki/Исследования/`):** Принадлежит AI и поддерживается им. Синтезирует источники в рабочие документы.
- **Схема (`AGENTS.md`):** Конфигурирует поведение AI и структуру wiki.
- **Логирование:** Каждое значимое изменение в wiki должно быть зафиксировано в `wiki/log.md`.

## Технологический стек и архитектура

- **Frontend:** React 19 (SPA) + TypeScript + Vite.
- **Стилизация:** Tailwind CSS v4 (используя `@tailwindcss/vite`).
- **Управление состоянием:** Zustand (`src/stores/`).
- **Backend:** Firebase Auth & Firestore (`src/lib/firebase.ts`).
- **Роутинг:** React Router v7.
- **Дизайн-система:** Высококонтрастный стиль "ClickHouse" (Желтый #faff69 на Черном #0a0a0a). См. `DESIGN.md`.
- **Сервисы:** Логика взаимодействия с Firebase находится в `src/lib/*Service.ts`.

## Рабочие процессы разработки

### Сборка и запуск
- **Установка:** `npm ci`
- **Разработка:** `npm run dev`
- **Сборка:** `npm run build` (включает проверку TypeScript и продакшн-сборку).
- **Предпросмотр:** `npm run preview`

### Тестирование (ОБЯЗАТЕЛЬНО)
**Задача считается невыполненной без прохождения тестов.**

#### Юнит-тестирование (Vitest)
- **Команда:** `npm test` (однократно) или `npm run test:watch`.
- **Расположение:** Файлы `*.test.ts(x)` рядом с исходным кодом.
- **Конвенции:**
  - Тестируйте поведение, а не реализацию.
  - Используйте `userEvent` для взаимодействий.
  - Имитируйте (mock) Firebase и сторы на уровне модулей при тестировании компонентов или сторов.
  - Используйте фабричные функции из `src/test/mocks/` для тестовых данных.
  - Имена тестов на русском языке.

#### E2E тестирование (Playwright)
- **Команда:** `npm run test:e2e:full` (запуск тестов + очистка).
- **Расположение:** `e2e/*.spec.ts`.
- **Конвенции:**
  - Обязательно для новых страниц или критических сценариев (Login, CRUD).
  - Используйте Page Object Model (POM) в `e2e/models/`.
  - Используйте кастомную фикстуру `test` из `e2e/fixtures.ts`.
  - Используйте `test@vitalik.dev` для статических тестов; `test-${Date.now()}@vitalik.dev` для тестов регистрации.
  - Очистка (cleanup) обязательна после прогона для поддержания чистоты Firebase.

### Стиль кода
- **Компоненты:** Функциональные компоненты с TypeScript интерфейсами для пропсов.
- **Стилизация:** Предпочитайте семантические классы Tailwind инлайновым стилям. Используйте цветовые токены "ClickHouse" (`bg-canvas`, `text-ink`, `text-yellow`).
- **Zustand:** Держите сторы сфокусированными и используйте паттерн `useAuthStore.setState` для обновлений вне экшенов, если это необходимо.
- **Firebase:** Всегда используйте сервисный слой; избегайте прямых вызовов Firestore/Auth из компонентов.

## Ключевые файлы и директории
- `GEMINI.md`: Этот файл (инструкции репозитория).
- `AGENTS.md`: Рабочий процесс AI и схема Wiki.
- `DESIGN.md`: Детальная спецификация дизайн-системы.
- `src/stores/`: Сторы состояний Zustand.
- `src/lib/`: Конфигурация Firebase и сервисный слой.
- `wiki/`: Документация проекта и исследования.
- `e2e/`: E2E тесты Playwright и модели.
